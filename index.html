在c++中，如果将字符串赋值给char*，或者将函数参数类型写为char*，传参时却用字符串，会遇到这个问题。

导致错误的原因是赋值操作等号两边的类型不同，等号右边的”Hello”是一个不变常量，在c++中叫作string literal，类型为const char*，而等号左边的p的类型却是char*，如果强行赋值会把右边的常量强制类型转换成一个可变的字符指针，结果使得我们可以修改这个常量，容易引发问题，因此这种写法在c++中已弃用。

const char*和char const*是等价的，它们与char* const不同。

这几个概念可以用”把声明从右向左读“的方法速记，即：

​ char * const p;: p is a const pointer to char;

​ const char * p;: p is a pointer to const char;

​ char const * p;: p is a pointer to const char;

第二句和第三句相同是因为一条规则：

​ const binds left, unless there is nothing on the left, then it binds right

​ const 是左结合的，左侧为空，则右结合
